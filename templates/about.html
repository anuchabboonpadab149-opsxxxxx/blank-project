กลับสู่แดชบอร์ด

สรุประบบ

นี่คือเครื่องสร้างคอนเทนต์อัตโนมัติ 24/7 ในธีม “จัสมินชอบกินแซลมอน” พร้อมสตรีมเหตุการณ์แบบเรียลไทม์ สร้างเสียง/รูป/วิดีโอ และกระจายไปยังแพลตฟอร์มโซเชียลที่ตั้งค่าไว้

  • ทำงานอัตโนมัติทุกวินาที (ปรับได้) และเก็บสถิติทุกนาที
  • เนื้อหาเน้นซาชิมิ/โชยุ/วาซาบิ/โอมากาเสะ (ปรับคำได้ใน Settings)
  • พร้อมแดชบอร์ดแสดงผลสด, ลิงก์สื่อล่าสุด และ API ครบ

เริ่มต้นใช้งาน

  1. ตั้งค่าไฟล์ .env ด้วยคีย์จริงของแพลตฟอร์มที่ต้องการ (Twitter/Facebook/LinkedIn/LINE/Telegram/Discord/Instagram/Reddit/Mastodon/Twitter Ads)
  2. รันด้วย Docker: docker compose up -d --build
  3. เปิดหน้าแดชบอร์ด: / และตรวจสุขภาพที่ /healthz

หมายเหตุ: คีย์จริงควรจัดการผ่าน .env หรือระบบ Secret Manager ของโครงสร้างพื้นฐานเท่านั้น

API ที่มีให้

  • /events — สตรีมเหตุการณ์ (SSE)
  • /api/latest, /api/feed, /api/recent
  • /api/config (GET/POST), /api/reload-schedule, /api/post-now
  • /api/nodes (GET), /api/nodes/register (POST), /api/nodes/heartbeat (POST), /api/nodes/summary (GET)
  • /media/... — ให้บริการไฟล์เสียง/รูป/วิดีโอ
  • /healthz — สุขภาพระบบ

ความปลอดภัย

  • หลีกเลี่ยงการใส่คีย์ผ่าน UI ใช้ .env หรือ Secret Manager แทน
  • ระบบออกแบบให้ทนทานต่อการล้มเหลว: โพสต์ล้มเหลวก็ยังมีสื่อและเหตุการณ์อัปเดตทุกวินาที
  • ตั้งค่า ADS_SIMULATION=false เฉพาะเมื่อพร้อมยิงโฆษณาจริงและงบครบ
